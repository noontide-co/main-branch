# Solved

Problems figured out. Don't re-solve.

## Why

When you figure out something non-obvious, capture it. Next time = instant answer.

## Format

```markdown
---
date: 2026-01-13
problem: Short problem statement
tags: [tag1, tag2]
---

# Problem Title

## Problem
[What was happening]

## Root Cause
[Why it was happening]

## Solution
[How you fixed it]

## Prevention
[How to avoid next time]
```

## Naming

`YYYY-MM-DD-problem-slug.md`

Example: `2026-01-13-shopify-webhooks-not-firing.md`

## What Goes Here

- Debugging sessions that took time
- Configuration that wasn't obvious
- Workarounds for tool limitations
- "Gotchas" you'll forget
